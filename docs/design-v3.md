## Problem Statement

People fail ambitious goals not because they lack resources, but because they lack structure AND execution support.

- Structure: AI decomposes goals into phases, generates weekly schedules, coaches daily.
- Execution: AI creates the actual deliverables... flashcards, research reports, presentations, study materials.

Example: "Learn English" doesn't just produce a schedule. The AI creates custom practice exercises, generates flashcard decks, compiles research into study materials. "Prepare Q2 presentation" doesn't put "make slides" on a todo list. The AI generates the actual PowerPoint.

The product category: **"AI goal agent that plans, coaches, AND does the work."**

## Target Users

Three user segments, in priority order:

1. **AI-anxious professionals** — Fear of being left behind by AI trends. Want simple AI tools that make them feel capable, not overwhelmed. Need: safety, simplicity, approachability.
2. **Overwhelmed professionals** — Too many things to handle, need AI to boost efficiency. Need: speed, directness, results.
3. **Aspiring entrepreneurs / independent builders** — Have ideas, want an AI team to plan and execute alongside them. Need: team feel, control, agency.

**Initial wedge:** General-purpose AI goal agent, any goal type. First real user signal is in language learning for professionals. Target market: US and Europe (not China, due to competitive copy-speed risk).

## Demand Evidence

- One real user: Chinese professional immigrant in the US, working at a US company
- She uses the Goals system for learning English: structured phases, weekly schedules, AI coaching
- She was attracted to: (1) structured schedule, (2) todo lists, (3) organized phases
- Before this tool: YouTube, tutor, English books, ChatGPT for word lookups. Plenty of resources, zero structure.
- Her motivation: English critical for daily work and career advancement

**New hypothesis:** If the AI also created her learning materials (flashcard decks, conversation scripts, pronunciation exercises), she would get even more value. The AI removes the "meta-work" of finding and organizing resources.

## Status Quo

Target users currently:
- Use generic AI chat (ChatGPT, Claude) for answers, but get no structure or follow-through
- Use domain-specific apps (Duolingo, fitness apps) that are structured but narrow, can't handle arbitrary goals
- Use productivity tools (Notion, Todoist) where they must create the plan AND the materials themselves
- Hire human coaches/tutors who are expensive and not available 24/7

**The gap:** No tool provides structure + coaching + AI execution for arbitrary goals. The user shouldn't have to create their own study materials, research their own trip itinerary, or build their own presentation. The AI should do the grunt work.

## Key Strategic Decisions (from founder Q&A)

### 1. UI: Dashboard-first, not game-first

The prior MyAIOffice had a full Pixi.js isometric office. This is replaced by:
- **Core experience = Dashboard** (fast, direct, professional). Goal list, weekly plan, daily tasks, coaching messages. This is where users spend 90% of their time. Mobile-friendly, loads fast.
- **Emotional layer = Lightweight gamification** (no Pixi.js). Agent avatars with status indicators, confetti on task completion, streak flames, personality in AI responses. Implemented with React + CSS animations, not Canvas.
- **Optional: Isometric office view** deferred. Can be added as a "View my office" easter egg if users want it. Not blocking MVP.

**Rationale:** User segment 1 (AI-anxious) needs approachability, not a game engine. Segment 2 (efficiency) needs speed, not animation. Segment 3 (entrepreneurs) gets "team feel" from agent personas, not from watching sprites walk.

### 2. Single LLM, multiple roles (not multiple employees)

- **One LLM:** Claude (Sonnet for planning, Haiku for daily coaching)
- **Multiple roles via system prompt switching:** Coach (daily, default), Researcher (web search tasks), Creator (file generation tasks), Challenger (weekly review, pushback). Users don't choose roles. The system auto-assigns based on task type.
- **Multi-LLM is infrastructure, not a feature:** OpenClaw-style LLM routing happens behind the scenes for cost optimization and fallback. Users never see it.
- **Phase 2:** If data shows users want "team feel," wrap roles in different employee personas (names, avatars). This is a UI change, not an architecture change.

### 3. Web-only product (no PC/Electron needed)

Claude API's code_execution tool provides server-side sandboxed code execution. This eliminates the need for OpenClaw, VPS, or desktop app:
- Python runs in Anthropic's sandbox (pre-installed: python-pptx, openpyxl, pandas, matplotlib, pillow, etc.)
- Web search and web fetch are built-in Claude API tools
- Files are returned via Files API (file_id → download)
- Code execution is **free** when used with web search or web fetch

**This means the entire product is one URL.** No VPS, no OpenClaw, no Electron.

### 4. Mobile web is essential (not optional)

A daily coaching/checklist product without mobile access dies by day 3. The web app is mobile-first responsive:
- **PC = Planning mode** (create goals, edit phases, review weekly plans, download files). Used 1-2x/week, 10-20 min.
- **Mobile = Execution mode** (check off tasks, read coaching messages, track streak). Used 3-5x/day, 10-30 sec each.
- Both are the same URL, same codebase, responsive Tailwind breakpoints.

### 5. English only for MVP

Target market is US/EU. i18n deferred until user data shows demand. Browser translation covers edge cases.

## How It Works (User Flow)

### Goal Creation (Conversational)
1. First-time user signs up → lands directly in coaching chat (no empty state)
2. AI Coach asks: "What goal are you working toward?"
3. User types: "I want to speak English like a native speaker"
4. AI asks follow-up questions (current level, time available, deadline, motivation)
5. AI generates 3-6 phases with milestones (streaming, phases appear one-by-one)
6. User sees a confirmation card. Can edit, reorder, rename, add/remove phases.
7. User confirms → goal + phases saved to database

### Weekly Planning
1. AI generates this week's schedule with suggested time slots (9:00-9:30 AM format)
2. Each task is classified:
   - **USER_ACTION** — Only the user can do this ("Practice speaking idioms, 30 min")
   - **AI_EXECUTABLE** — AI can produce a deliverable ("Create flashcard deck of 50 business idioms")
   - **AI_ASSISTED** — AI helps, user finishes ("Draft email to boss requesting English training budget")
3. AI_EXECUTABLE tasks show a "Let AI do this" button

### Task Execution (via Claude code_execution)
1. User clicks "Let AI do this" on an executable task
2. Next.js API route calls Claude API with code_execution + web_search tools
3. Claude decides what to do:
   - Research task → web_search + compile findings into structured document
   - Create flashcards → code_execution (Python generates JSON/HTML)
   - Generate PPT → code_execution (python-pptx in sandbox)
   - Create Excel → code_execution (openpyxl in sandbox)
   - Generate chart → code_execution (matplotlib in sandbox)
4. Generated files returned via Files API → stored in Supabase Storage
5. Result appears on the task card. User views/downloads the deliverable.
6. Content that works better inline (flashcards, study materials) renders directly in the app, not as downloadable files.

### Daily Experience
1. User opens app (phone or desktop)
2. Sees today's tasks with time estimates and execution status
3. Checks off completed tasks
4. Coaching message appears contextually: "You completed 4/5 tasks yesterday. The one you skipped was the hardest. Try breaking it into 10 minutes before lunch today."
5. Streak counter updates
6. "Talk to coach about this" button on any task/phase for contextual feedback

### Notifications
- **Web Push** (primary): Morning nudge (8AM) + evening check-in (8PM), user timezone
- **Email nudge** (fallback): Same content via Resend for users on iOS or who decline Web Push
- **Weekly progress email**: Sunday evening, AI-generated summary of the week

## LLM Cost Model

All AI capabilities via Claude API. No VPS. No third-party execution services.

| Interaction | Frequency | Model | Tokens (in/out) | Cost/call | Monthly |
|---|---|---|---|---|---|
| Goal decomposition | 1x/goal | Sonnet | 1K/2K | $0.033 | $0.033 |
| Weekly plan generation | 4x/mo | Sonnet | 1.5K/2K | $0.034 | $0.136 |
| Daily coaching | 25x/mo | Haiku | 800/400 | $0.0005 | $0.013 |
| Weekly review | 4x/mo | Sonnet | 2K/1.5K | $0.029 | $0.116 |
| Task execution (code_exec) | 8x/mo | Sonnet + tools | 2K/4K | $0.066 | $0.528 |
| Web search (with code_exec) | 4x/mo | Sonnet | included | FREE | $0.00 |

**Monthly cost per active user: ~$0.83**

Code execution is FREE when used alongside web_search or web_fetch. This makes the execution layer nearly cost-free for research + file generation workflows.

At $15/mo subscription: ~94% gross margin. No VPS cost to amortize.

**Infrastructure cost: $0/mo** (Vercel free/pro tier + Supabase free tier). First dollar of revenue is nearly pure margin.

## Phased Rollout

### Phase 1: Planning + Coaching (ship first, CC: ~1 week)

Build and deploy the full web app WITHOUT execution:
- Conversational goal creation (chat with AI coach, streaming)
- Phase decomposition + weekly planning with time slots
- Daily task checklist with check-off
- Coaching messages (contextual, on app open)
- Streak counter + confetti animation
- Web Push notifications + email nudge fallback
- Weekly progress email
- Founder analytics dashboard (5 SQL queries)
- "Talk to coach" contextual feedback button
- Mobile-first responsive design
- Supabase Auth (email + Google OAuth)

Deploy to users. Start validation.

### Phase 2: AI Execution (add while Phase 1 users test, CC: ~3-5 days)

Add execution layer via Claude API code_execution:
- Task classification (USER_ACTION / AI_EXECUTABLE / AI_ASSISTED)
- "Let AI do this" button on executable tasks
- Claude code_execution for file generation (PPT, Excel, charts, documents)
- Claude web_search for research tasks
- Results stored in Supabase Storage or rendered inline
- execution_jobs table + daily_tasks columns via migration
- Execution failure handling (retry, edit, regenerate)

### Phase 3: Role-based Coaching + Gamification (after validation)

- Auto-assigned roles (Coach, Researcher, Creator, Challenger) with distinct system prompts
- If users want "team feel," wrap roles in employee personas (names, avatars)
- Lightweight gamification (achievement badges, milestone celebrations)
- Data export (JSON export endpoint for user portability)

## Auth & Security

- **Auth:** Supabase Auth with email/password + Google OAuth. No WeChat (targeting US/EU).
- **RLS:** Row Level Security on every table, scoped to auth.uid().
- **Admin:** Founder analytics page protected by email check (FOUNDER_EMAIL env var).
- **Degraded mode:** localStorage caches last-known weekly plan and daily tasks. Users can check off tasks even when Claude/Supabase are down.
- **Privacy:** No logging of goal content beyond what Supabase stores. Claude API standard data retention applies.

## Open Questions

1. **Execution quality:** What if AI-generated flashcards are wrong? Need "regenerate" and "edit" affordances for every deliverable.
2. **File storage limits:** Supabase free tier has 1GB storage. Enough for MVP (<50 users). Monitor.
3. **Claude code_execution latency:** PPT generation may take 30-60s. Need a loading state with progress indication. Streaming not available for code_execution results.
4. **Vercel function timeout:** code_execution calls may exceed Vercel free tier's 10s limit. May need Vercel Pro (60s) or use streaming endpoints.

## Success Criteria

- **Week 2:** 5+ users with at least one goal, completing daily tasks
- **Week 4:** 3+ users still active. Coaching messages read rate >50%.
- **Week 6:** One user offers to pay or asks about pricing.
- **Week 8 (Phase 2 live):** At least 10 AI-executed tasks completed. >50% of active users try execution.
- **Month 3:** Revenue > $0. Clear signal on which execution types are most valued.

**Minimum viable signal:** 5 WAU by week 8 with >50% using execution features. Below this, re-evaluate whether execution adds value over coaching alone.
